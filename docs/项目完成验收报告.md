## 交付建议

- 提交前执行 `git diff --check` 和必要 shard。
- 提交前优先跑 `run_pr_release_gate.ps1 -Stage pr`；合并前跑 `-Stage merge`。
- 发布审查先跑 `run_pr_release_gate.ps1 -Stage release -Mode preflight`；发布前再跑 `-Stage release -ContinueOnFailure`，然后执行 release gate、release live CI artifacts 和 full live validation 证据链。
- 客户试用前先跑 `run_pr_release_gate.ps1 -Stage customer -Mode preflight`；正式试用演练跑 `-Stage customer -ContinueOnFailure`，把客户能理解的安装、自检、核心非 live 证据和预检结果放在同一轮演练里。
- fixed runner 上线前确认 `PowerShell / Python 3.12 / Godot / Chrome 或 Edge / deployment/release_live_runner_profile.json / config.yaml / release-live-gates workflow_dispatch token` 均在 runner baseline 或 dispatch preflight 中显示为 passed；`browser_path` 只能作为 preflight warning，真实 browser lane 发布前必须显式配置。
- 后续优化方向应优先放在 release/promotion 测试性能，而不是继续增加路线图功能。
