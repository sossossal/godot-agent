## 交付建议

- 提交前执行 `git diff --check` 和必要 shard。
- 提交前优先跑 `-Profile quick`；发布前跑 `-Profile release -ContinueOnFailure` 并保留 `logs/reports/non_live_validation_shards.json`。
- 发布前先跑 `run_release_live_gates_locally.ps1 -Preflight`，再执行 release gate、release live CI artifacts 和 full live validation 证据链。
- 客户试用前跑 `-Profile customer`，把客户能理解的安装、自检、核心非 live 证据和预检结果放在同一轮演练里。
- 后续优化方向应优先放在 release/promotion 测试性能，而不是继续增加路线图功能。
