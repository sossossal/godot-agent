# IDE 集成演示指南

## 准备工作

```bash
cd D:\新建文件夹\godot-agent
pip install -r requirements.txt
python -m agent_system.cli doctor
```

如果需要真实场景创建、场景测试或导出，请先配置 `config.yaml` 中的 Godot 路径。

## 演示 1: 命令行代理

```bash
python ide_integration/vscode_agent.py "生成 2D 玩家移动脚本"
python ide_integration/vscode_agent.py "生成血量系统代码"
python ide_integration/vscode_agent.py "为敌人创建巡逻 AI"
```

期望结果:

- 终端显示任务状态
- 如果产物是脚本，会显示语法高亮代码
- 实际文件路径来自 `Task.artifacts`
- `.agent_result.json` 会保存最新任务结果

## 演示 2: Godot 控制器

```bash
python ide_integration/godot_controller.py status
python ide_integration/godot_controller.py open
python ide_integration/godot_controller.py create TestScene 2D
```

## 演示 3: 交互式 CLI

```bash
python -m agent_system.cli chat
```

可直接输入:

```text
生成玩家移动脚本
创建一个名为 DemoScene 的 2D 场景
为敌人创建巡逻 AI
导出 Web 项目
```

## 演示 4: 多步骤工作流

```bash
python -m agent_system.cli plan "创建一个场景并生成脚本并运行测试"
python -m agent_system.cli run "创建一个场景并生成脚本" -y
```

## 常见问题

### `No module named 'yaml'`

```bash
pip install -r requirements.txt
```

### 场景创建失败

确认:

- Godot 已安装
- `config.yaml` 的 `executable_path` 正确，或已设置 `GODOT` / `GODOT_EXE` / `GODOT_PATH`，或 Godot 已加入 `PATH`
- `project_path` 指向真实 Godot 项目

### 生成的代码在哪里

默认位于:

- `scripts/`
- `scripts/ai/`

也可以从 `Task.artifacts` 读取路径。
