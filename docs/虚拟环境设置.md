# 🐍 虚拟环境设置指南

## 🚀 自动设置 (推荐)

我们提供了自动化脚本,一键完成所有设置!

### Windows

#### 方式 1: 双击运行
直接双击 `setup_venv.bat` 文件

#### 方式 2: 命令行
```cmd
setup_venv.bat
```

#### 方式 3: PowerShell
```powershell
python setup_venv.py
```

---

### Linux / macOS

```bash
# 添加执行权限
chmod +x setup_venv.sh

# 运行设置脚本
./setup_venv.sh
```

或直接运行 Python 脚本:
```bash
python3 setup_venv.py
```

---

## ✅ 设置完成后

虚拟环境创建在 `venv/` 目录中,所有依赖已安装。

### 激活虚拟环境

**Windows PowerShell:**
```powershell
.\activate.ps1
```

**Windows CMD:**
```cmd
activate.bat
```

**Linux/macOS:**
```bash
source activate.sh
```

### 退出虚拟环境

在任何平台上都使用:
```
deactivate
```

## 📦 手动设置 (如果自动设置失败)

### 步骤 1: 创建虚拟环境

**Windows:**
```cmd
python -m venv venv
```

**Linux/macOS:**
```bash
python3 -m venv venv
```

### 步骤 2: 激活虚拟环境

**Windows PowerShell:**
```powershell
.\venv\Scripts\Activate.ps1
```

**Windows CMD:**
```cmd
venv\Scripts\activate.bat
```

**Linux/macOS:**
```bash
source venv/bin/activate
```

### 步骤 3: 安装依赖

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

## 💡 常见问题

### Q: 为什么需要虚拟环境?

虚拟环境可以:
- ✅ 隔离项目依赖,避免版本冲突
- ✅ 保持系统 Python 环境干净
- ✅ 方便项目迁移和部署
- ✅ 便于团队协作

### Q: 虚拟环境在哪里?

创建在项目根目录的 `venv/` 文件夹中

### Q: 如何删除虚拟环境?

先退出虚拟环境:
```bash
deactivate
```

然后删除文件夹:
```bash
# Windows
rmdir /s venv

# Linux/macOS
rm -rf venv
```

### Q: PowerShell 提示无法运行脚本?

如果遇到执行策略错误,以管理员身份运行 PowerShell:
```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q: 每次都要激活虚拟环境吗?

是的,每次在新的终端窗口中使用项目时都需要激活。这是 Python 虚拟环境的标准做法。

## 🔧 IDE 集成

### VSCode

VSCode 会自动检测虚拟环境。确保:
1. 打开项目文件夹
2. 选择 Python 解释器: `Ctrl+Shift+P` → "Python: Select Interpreter"
3. 选择 `./venv/Scripts/python.exe` (Windows) 或 `./venv/bin/python` (Linux/macOS)

VSCode 的集成终端会自动激活虚拟环境!

### PyCharm

PyCharm 也会自动检测 `venv/` 目录:
1. File → Settings → Project → Python Interpreter
2. 选择 venv 中的 Python

## 📝 最佳实践

1. **总是在虚拟环境中工作**
   ```bash
   # 激活环境
   source activate.sh  # 或 activate.bat
   
   # 工作...
   python examples/interactive_demo.py
   
   # 完成后退出
   deactivate
   ```

2. **定期更新依赖**
   ```bash
   pip install --upgrade pip
   pip install --upgrade -r requirements.txt
   ```

3. **添加到 .gitignore**
   虚拟环境已自动添加到 `.gitignore`,不会提交到 Git
