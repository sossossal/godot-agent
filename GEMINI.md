# Godot 开发指令 (Gemini CLI 版)

本工作区已集成 Godot Agent MCP 工具。当需要进行 Godot 游戏制作、场景构建或脚本编写时，请遵循以下规约：

## 核心工具
- **`godot_make`**: 执行一切具体的开发动作。它会自动规划、执行并更新项目蓝图。
- **`godot_status`**: 在开始大的功能前，调用此工具同步当前的“项目灵魂”状态。
- **`godot_capture`**: 获取当前编辑器的视觉快照。

## 交互流程
1. **同步状态**: 接到指令后，先运行 `godot_status` 了解项目现状（已有哪些功能，当前游戏类型）。
2. **执行指令**: 使用 `godot_make` 下发用户的具体需求。
3. **视觉反馈**: 执行后若产生场景变更，建议跟一个 `godot_capture` 展示成果。

## 注意事项
- 尽量将复杂需求通过 `godot_make` 的自然语言接口传达，系统内部会自动拆解技能链。
- 若执行结果返回 `FAILED`，可调用 `godot_make` 进行“自修复”或“审计逻辑”。
